# CogApp — Architecture Map

## App Shape

CogApp is a Vue 3, TypeScript, Vite single-page app for managing college applications. It has Firebase-backed accounts and cloud sync, while still keeping a local browser backup for fast access and offline resilience.

## Root Files

| File | What It Does |
|------|-------------|
| `index.html` | Browser entry point and PWA metadata |
| `package.json` | Scripts and dependencies |
| `vite.config.ts` | Vite build configuration |
| `public/manifest.json` | Installable-app metadata |
| `public/sw.js` | Production service worker for same-origin caching |

## Main Source Areas

| Area | Purpose |
|------|---------|
| `src/main.ts` | Creates Vue app, registers production service worker, runs Firebase/cloud sync |
| `src/App.vue` | App shell with sidebar for logged-in pages and guest layout for auth pages |
| `src/router/index.ts` | Routes and auth/onboarding guards |
| `src/stores/` | Pinia stores for user data domains |
| `src/services/` | Firebase, cloud sync, storage, backup, matching, and API helpers |
| `src/views/` | Page-level UI |
| `src/components/` | Shared UI components |

## Auth And Data Flow

1. `userStore.ts` listens to Firebase Auth and exposes the logged-in user.
2. `router/index.ts` waits for Firebase auth before allowing protected routes.
3. Stores save user data through localStorage keys generated by `getUserKey(...)`.
4. `main.ts` watches selected user localStorage keys and syncs them to Firestore.
5. Cloud pulls only replace local data when the cloud copy is newer.
6. `storageService.ts` migrates larger file data to IndexedDB per user.

## Application Command Center

- `applicationStore.ts` owns application statuses, per-college checklists, tasks, and recommendation records.
- `ApplicationCommandCenter.vue` presents Applications, Tasks, Recommendations, and the app's full Calendar over the same data.
- Essay and recommendation checklist rows are derived from their source stores instead of duplicated manual statuses.
- Default checklist dates inherit the College List deadline; only explicit overrides are stored separately.
- The `command-center` user key is included in Firestore sync and JSON backup/restore.

## Key Services

| File | What It Does |
|------|-------------|
| `firebaseService.ts` | Firebase app setup, auth helpers, Firestore read/write helpers |
| `cloudStorage.ts` | Cloud load/save wrapper with local fallback and cache |
| `storageService.ts` | One-time per-user IndexedDB file migration |
| `fileStorage.ts` | Higher-level IndexedDB file helper |
| `dataBackup.ts` | JSON export/import of user data |
| `scholarshipMatcher.ts` | Scholarship scoring and scam-warning heuristics |

## Important Safety Notes

- Firebase Auth, not localStorage, decides whether the user is logged in.
- Onboarding and custom Brag Sheet tabs are per-user data.
- The service worker should stay production-only to avoid confusing development with stale files.
- Generated `.js` files should not live in `src`; TypeScript and Vue files are the source of truth.
